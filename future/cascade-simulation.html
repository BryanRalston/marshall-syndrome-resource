This simulation uses estimated parameters to illustrate how the 8 self-amplifying cascades in Marshall syndrome interact and how drug interventions might reduce total disease burden. The coupling strengths, growth rates, and drug effects shown here are approximations based on published research, not experimentally validated values. This tool is meant to build intuition, not to predict actual treatment outcomes. Always discuss treatment decisions with your medical team.

The Unified Cascade Model identifies 8 self-amplifying destructive loops triggered by the COL11A1 mutation. These loops feed each other through a network of connections, meaning the disease is not 8 separate problems but one interconnected system. This simulation models that system as coupled differential equations, letting you see how cascade activity evolves over time and how different drug interventions change the trajectory.

Understanding the Math (Plain Language)

This simulation tracks 8 numbers, one for each cascade. Each number represents how active that cascade is, on a scale from 0 (completely inactive) to 1 (maximally active).

How each cascade changes over time: At every time step, each cascade's activity level is updated by three forces:

  • Self-amplification: Active cascades tend to grow (like a fire that feeds itself), but they saturate — they can't grow past 1.0. This is modeled as logistic growth.
  • Cross-feeding: Other cascades push this one up. For example, the senescence cascade (Loop 3) feeds the MMP cascade (Loop 1), so when senescence is high, MMP activity also increases. The strength of each connection is based on the biological relationships described in the Unified Model.
  • Decay: Without ongoing stimulation, cascades naturally decline (the body has some repair capacity).
  • Drug effects: When a drug is active, it reduces cascade activity by a percentage specific to each drug-cascade pair. This is subtracted at each time step.

The solver: Because these equations are coupled (each cascade depends on all the others), we use a 4th-order Runge-Kutta integrator — a standard numerical method that computes each time step accurately by sampling the rate of change at four points within each step. This avoids the numerical instability that simpler methods produce with interconnected systems.

What this shows: The key insight is that cascade interconnections create outsized effects from intervention. Reducing one cascade doesn't just lower that cascade — it reduces the input to every cascade it feeds, which reduces their inputs to others, and so on. This is why multi-target, early intervention has disproportionately large effects compared to single-target or late intervention.
